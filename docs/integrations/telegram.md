# Telegram-бот

SanCRM поддерживает интеграцию с Telegram через собственного бота. Бот позволяет автоматически уведомлять клиентов о смене статуса ремонта, а сотрудникам — получать оперативную информацию в групповых чатах.

## Подключение бота

Для работы интеграции необходимо создать Telegram-бота через BotFather и указать полученный токен в настройках SanCRM. После сохранения токена система автоматически проверит подключение и активирует бота.

{% note info %}

Один бот подключается к одному аккаунту SanCRM. Если у вас несколько филиалов, все они используют одного и того же бота, но уведомления можно настроить отдельно для каждого филиала.

{% endnote %}

## Команды бота

Бот поддерживает набор команд, с помощью которых клиенты могут самостоятельно взаимодействовать с сервисным центром. Основные команды позволяют узнать текущий статус ремонта, получить информацию о стоимости и сроках.

<!-- screenshot: пример диалога с ботом — ввод команды и ответ со статусом ремонта -->

## Привязка клиента к Telegram

Клиент привязывается к Telegram-боту по номеру телефона или через уникальную ссылку. После привязки бот может отправлять персональные уведомления о ремонтах, связанных с этим клиентом.

{% note info %}

Привязка происходит автоматически, когда клиент впервые пишет боту и делится своим контактом. Также можно отправить клиенту ссылку для привязки вручную из карточки клиента.

{% endnote %}

<!-- screenshot: карточка клиента с индикатором привязки Telegram и кнопкой отправки ссылки -->

## Уведомления клиентов

Система автоматически отправляет клиенту сообщения в Telegram при смене статуса ремонта. Шаблоны уведомлений можно настроить: задать текст для каждого перехода статуса и включить или отключить отдельные типы уведомлений.

<!-- screenshot: настройка шаблонов уведомлений — список статусов с переключателями и полями для текста -->

## Групповые чаты для сотрудников

Бот можно добавить в групповой чат Telegram, где он будет публиковать уведомления для сотрудников — например, о поступлении нового ремонта, изменении статуса или комментарии от клиента. Это позволяет мастерам оперативно реагировать без постоянного нахождения в системе.

{% note info %}

Для работы групповых уведомлений бот должен быть добавлен в чат и иметь права на отправку сообщений. Идентификатор группового чата указывается в настройках интеграции.

{% endnote %}

<!-- screenshot: пример уведомления в групповом чате Telegram — новый ремонт с кратким описанием -->

## Настройка типов уведомлений

В разделе настроек Telegram-бота можно выбрать, какие события будут отправляться клиентам, а какие — в групповой чат сотрудников. Для каждого события доступен переключатель и возможность редактирования текста сообщения.

<!-- screenshot: таблица событий с переключателями для клиентского и группового канала -->

## Устранение неполадок

Если уведомления не доставляются, необходимо проверить корректность токена, статус привязки клиента и доступность Telegram API. В логе интеграции фиксируются все попытки отправки с указанием результата и возможных ошибок.
