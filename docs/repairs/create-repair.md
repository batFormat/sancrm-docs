# Создание ремонта

Создание заказа на ремонт — первый шаг в работе с устройством клиента. Форма создания собирает всю необходимую информацию для дальнейшей работы мастера.

<!-- screenshot: кнопка создания нового ремонта на панели инструментов -->

## Открытие формы создания

Новый ремонт можно создать несколькими способами из разных частей интерфейса.

- Кнопка «Новый ремонт» на главной панели.
- Быстрое создание из карточки клиента.

<!-- screenshot: форма создания нового ремонта -->

## Выбор клиента

При создании ремонта необходимо указать клиента — владельца устройства. Система поддерживает поиск по существующей базе и создание нового клиента на лету.

- **Поиск существующего клиента** — по имени, телефону или email с автоподсказками.
- **Создание нового клиента** — быстрая форма без необходимости покидать страницу ремонта.
- **Автозаполнение** — если клиент найден, его данные подставляются автоматически.

{% note info %}

Если клиент обращается повторно, система покажет историю его предыдущих ремонтов. Это помогает учитывать контекст при приёме устройства.

{% endnote %}

## Информация об устройстве

Блок устройства описывает, что именно принято в ремонт. Корректное заполнение этих полей важно для идентификации техники.

- **Тип устройства** — выбор из справочника (смартфон, ноутбук, планшет и т.д.).
- **Производитель и модель** — выбор из каталога или ручной ввод.
- **Серийный номер / IMEI** — для точной идентификации устройства.
- **Внешний вид и комплектация** — описание состояния устройства при приёме.

<!-- screenshot: блок информации об устройстве в форме создания -->

## Описание неисправности

Описание проблемы со слов клиента — ключевая информация для мастера, который будет проводить диагностику.

- **Описание проблемы** — текстовое поле для записи слов клиента.
- **Предварительный диагноз** — опциональное поле для приёмщика, если неисправность очевидна.
- **Вложения** — возможность прикрепить фотографии устройства при приёме.

{% note warning %}

Рекомендуется всегда фиксировать состояние устройства при приёме — внешние повреждения, комплектацию, наличие SIM-карт и карт памяти. Это поможет избежать спорных ситуаций при выдаче.

{% endnote %}

## Назначение исполнителя

При создании ремонта можно сразу назначить мастера, ответственного за выполнение работы.

## Предварительная стоимость

При создании ремонта можно указать предварительную оценку стоимости для клиента.

- **Ориентировочная стоимость** — сумма, озвученная клиенту при приёме.
- **Предоплата** — фиксация внесённой предоплаты.

## Сохранение и печать

После заполнения формы заказ сохраняется и система предлагает напечатать документы для клиента.

- **Сохранение** — заказ создаётся со статусом «Принят» (или другим начальным статусом).
- **Печать квитанции** — автоматическое предложение напечатать приёмную квитанцию.
- **Переход к карточке** — после сохранения открывается полная карточка ремонта.
