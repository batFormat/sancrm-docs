# Квитанции и документы

Модуль документов обеспечивает формирование и печать квитанций приёма на основе настраиваемых DOCX-шаблонов. Документы генерируются автоматически с подстановкой данных из карточки ремонта.

<!-- screenshot: список доступных документов в карточке ремонта -->

## Типы документов

Система поддерживает два типа документов для ремонтов.

### Квитанция приёма

Основной документ, подтверждающий приём устройства в ремонт. Выдаётся клиенту при оформлении заказа.

- Содержит информацию о клиенте, устройстве и описании неисправности.
- Включает данные организации и филиала.
- Формируется в формате PDF из DOCX-шаблона.

### Квитанция краткая

Компактная версия приёмной квитанции для быстрой печати.

<!-- screenshot: пример напечатанной приёмной квитанции -->

## Печать документов

Печать доступна из карточки ремонта. Система генерирует PDF-файл из DOCX-шаблона с подставленными данными.

- **Генерация PDF** — система заполняет шаблон данными из заказа и формирует PDF.
- **Предварительный просмотр** — просмотр сформированного документа перед печатью.
- **Скачивание** — документ можно скачать в формате PDF.

## Шаблоны документов

Каждый тип документа формируется по DOCX-шаблону, который загружается администратором.

- **Загрузка шаблона** — шаблон загружается как файл DOCX с переменными-плейсхолдерами.
- **Переменные** — в шаблонах используются переменные вида `${field}`, которые заменяются данными из заказа.
- **Один шаблон на тип** — для каждого типа документа загружается один шаблон.

{% note info %}

Настройка шаблонов документов описана в разделе [Шаблоны документов](../settings/templates.md). Для создания шаблона подготовьте DOCX-файл с переменными и загрузите его в систему.

{% endnote %}

{% note warning %}

Убедитесь, что шаблоны документов корректно настроены и содержат все необходимые реквизиты перед началом работы.

{% endnote %}
