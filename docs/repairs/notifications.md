# Уведомления клиентов

Система уведомлений информирует клиентов о ходе ремонта через SMS, email и Telegram. Автоматические уведомления повышают лояльность и снижают количество входящих звонков с вопросами о статусе.

<!-- screenshot: настройка уведомлений для статусов ремонта -->

## Каналы уведомлений

SanCRM поддерживает несколько каналов доставки уведомлений клиентам.

### SMS

Отправка текстовых сообщений на мобильный телефон клиента.

- Интеграция с SMS-провайдерами (настраивается в разделе [Интеграции](../integrations/)).
- Ограничение длины сообщения с учётом тарификации.
- Транслитерация для экономии символов (опционально).

### Email

Отправка электронных писем на email-адрес клиента.

- Поддержка HTML-шаблонов с фирменным стилем.
- Возможность прикрепления документов (квитанции, акты).
- Настройка адреса и имени отправителя.

### Telegram

Отправка сообщений через Telegram-бота.

- Клиент подключается к боту по ссылке или QR-коду.
- Поддержка форматированных сообщений.
- Возможность добавления кнопок для быстрых действий (например, «Согласовать стоимость»).

{% note info %}

Для работы уведомлений через Telegram необходимо настроить бота. Подробная инструкция в разделе [Интеграции](../integrations/).

{% endnote %}

## Автоматические уведомления

Автоматические уведомления отправляются при смене статуса ремонта. Для каждого перехода можно настроить отдельное уведомление.

- **Привязка к переходу** — уведомление срабатывает при переходе между конкретными статусами.
- **Выбор канала** — для каждого перехода выбирается один или несколько каналов доставки.
- **Задержка отправки** — отправка может быть отложена на указанное время.

## Шаблоны уведомлений

Каждое уведомление формируется по шаблону с поддержкой переменных. Шаблоны настраиваются для каждого канала и перехода.

- **Переменные** — подстановка данных из заказа: имя клиента, номер ремонта, статус, стоимость, дата готовности.
- **Условные блоки** — части текста, которые включаются только при выполнении условия.
- **Многоязычность** — поддержка шаблонов на нескольких языках (при необходимости).

Пример шаблона SMS:

```
Здравствуйте, {{client_name}}! Ваш ремонт №{{repair_number}} переведён в статус "{{status}}".
```

## Ручная отправка

Помимо автоматических уведомлений, можно отправить сообщение клиенту вручную из карточки ремонта.

- **Произвольное сообщение** — ввод текста вручную.
- **Из шаблона** — выбор из списка заготовленных шаблонов.
- **Выбор канала** — отправка через любой доступный канал.

## Приоритет каналов

Если у клиента указано несколько контактных каналов, система выбирает оптимальный по настроенному приоритету.

- Порядок приоритета задаётся администратором (например, Telegram > SMS > Email).
- Если предпочтительный канал недоступен, используется следующий по приоритету.
- Можно настроить отправку одновременно по нескольким каналам.

{% note warning %}

Отправка SMS тарифицируется провайдером. Контролируйте баланс SMS-шлюза, чтобы уведомления не прекращались в неожиданный момент. Система предупредит о низком балансе.

{% endnote %}

## История уведомлений

Все отправленные уведомления сохраняются в истории — как автоматические, так и ручные.

- Дата и время отправки.
- Канал доставки.
- Текст сообщения.
- Статус доставки (отправлено, доставлено, ошибка).
