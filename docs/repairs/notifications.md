# Уведомления

Система уведомлений позволяет информировать сотрудников о событиях, связанных с ремонтами, через Telegram-чат. События фиксируются в хронологии (timeline) карточки ремонта.

<!-- screenshot: хронология событий в карточке ремонта -->

## Telegram-уведомления для сотрудников

SanCRM отправляет уведомления в настроенный Telegram-чат при определённых событиях. Это позволяет мастерам и администраторам оперативно реагировать на изменения.

- **Не дозвонились** — если клиенту не удалось дозвониться, система отправляет уведомление в Telegram с ссылкой на ремонт.
- **Ссылка на ремонт** — уведомление содержит кнопку для перехода к карточке ремонта в CRM.

{% note info %}

Для работы Telegram-уведомлений необходимо настроить бота и указать ID чата в настройках. Подробная инструкция в разделе [Интеграции](../integrations/).

{% endnote %}

## Хронология событий (Timeline)

Все значимые события по ремонту фиксируются в хронологии карточки. Каждая запись содержит тип события, автора, дату и время.

Типы событий:

- **Смена статуса** — переход ремонта из одного статуса в другой.
- **Назначение мастера** — назначение или смена исполнителя.
- **Назначение ответственного** — назначение ответственного сотрудника.
- **Исходящий звонок** — фиксация звонка клиенту.
- **Исходящее SMS** — фиксация отправки SMS (при наличии интеграции).
- **Комментарий** — текстовый комментарий сотрудника.
- **Не дозвонились** — фиксация неудачной попытки связаться с клиентом.
- **Заказ запчастей** — фиксация факта заказа запчастей.

## Публичная проверка статуса

Клиенты могут самостоятельно проверить статус ремонта на публичной странице, не требующей авторизации. Подробнее — в разделе [Публичная проверка статуса](../integrations/public-status.md).
