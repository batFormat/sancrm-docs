# Филиалы

Если ваш сервисный центр имеет несколько точек обслуживания, раздел филиалов позволяет настроить каждую из них отдельно — с собственным адресом, графиком работы и набором сотрудников.

<!-- screenshot: список филиалов в настройках -->

## Создание филиала

Чтобы добавить новый филиал, нажмите кнопку создания и заполните основные сведения. Каждый филиал получает уникальное название, которое используется для фильтрации ремонтов и складских операций.

- **Название** — отображаемое имя филиала (например, «Центральный офис» или «Филиал на Ленина»).
- **Код** — короткий идентификатор для внутреннего использования.

<!-- screenshot: форма создания нового филиала -->

## Адрес и контакты филиала

Для каждого филиала укажите физический адрес и контактную информацию. Эти данные используются в печатных документах и могут отображаться клиентам при проверке статуса ремонта.

<!-- screenshot: поля адреса и контактов филиала -->

## График работы

Настройте расписание работы филиала по дням недели. График работы учитывается при расчёте ориентировочных сроков ремонта и отображается клиентам.

- Для каждого дня недели укажите время начала и окончания рабочего дня.
- Выходные дни можно отметить как нерабочие.

{% note info %}

График работы филиала не влияет на доступ сотрудников к системе — пользователи могут входить в SanCRM в любое время.

{% endnote %}

<!-- screenshot: настройка графика работы по дням недели -->

## Привязка сотрудников

Назначьте сотрудников, которые работают в данном филиале. Привязка к филиалу определяет, какие ремонты и складские остатки видит сотрудник по умолчанию.

- Один сотрудник может быть привязан к нескольким филиалам.
- Администраторы имеют доступ ко всем филиалам независимо от привязки.

<!-- screenshot: список сотрудников филиала с возможностью добавления -->

## Филиал по умолчанию

Один из филиалов можно отметить как филиал по умолчанию. Он будет автоматически подставляться при создании новых ремонтов и складских документов.

{% note warning %}

Удаление филиала невозможно, если к нему привязаны активные ремонты или складские остатки. Сначала перенесите данные в другой филиал.

{% endnote %}
