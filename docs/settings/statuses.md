# Статусы ремонта

Настройка статусов ремонта позволяет определить жизненный цикл заказа в вашем сервисном центре. Каждый статус отражает этап работы над устройством — от приёма до выдачи клиенту.

<!-- screenshot: страница настройки статусов с цветными плашками -->

## Создание статуса

Для добавления нового статуса задайте его название и параметры отображения. Статус будет доступен для использования в карточках ремонта и на канбан-доске.

- **Название** — отображаемое имя статуса (например, «В диагностике», «Ожидает запчасти»).
- **Системное имя** — уникальный идентификатор статуса для внутреннего использования.

<!-- screenshot: форма создания нового статуса -->

## Цвета статусов

Каждому статусу назначается цвет, который используется для визуального различия на канбан-доске, в списке ремонтов и в карточках заказов. Выберите цвет из палитры или укажите произвольный HEX-код.

<!-- screenshot: палитра выбора цвета для статуса -->

## Порядок отображения

Статусы отображаются на канбан-доске и в фильтрах в определённом порядке. Перетащите статусы в нужную последовательность, чтобы они отражали логику вашего рабочего процесса.

{% note info %}

Порядок статусов определяет расположение колонок на канбан-доске. Рекомендуется выстроить статусы слева направо — от приёма до выдачи.

{% endnote %}

<!-- screenshot: перетаскивание статусов для изменения порядка -->

## Статус по умолчанию

Один из статусов назначается статусом по умолчанию — он автоматически присваивается каждому новому ремонту при создании. Обычно это статус «Принят» или «Новый».

<!-- screenshot: переключатель статуса по умолчанию -->

## Допустимые переходы

Настройте, из какого статуса в какой разрешён переход. Ограничение переходов помогает соблюдать последовательность рабочего процесса и предотвращает случайный пропуск этапов.

- Для каждого статуса определите список статусов, в которые из него можно перейти.
- Если переходы не ограничены, из любого статуса можно перейти в любой другой.

{% note warning %}

Будьте внимательны при ограничении переходов. Слишком жёсткие правила могут затруднить работу сотрудников в нестандартных ситуациях. Рекомендуется оставить возможность возврата на предыдущие этапы.

{% endnote %}

<!-- screenshot: матрица допустимых переходов между статусами -->

## Финальные статусы

Отметьте статусы, которые являются финальными — они означают завершение ремонта (например, «Выдан» или «Отказ»). Ремонты в финальном статусе не отображаются в активных заказах и учитываются в статистике как завершённые.

<!-- screenshot: настройка финального статуса с флажком -->

## Удаление и архивация статусов

Статус, к которому привязаны существующие ремонты, нельзя удалить. Вместо удаления скройте неиспользуемый статус — он перестанет отображаться при выборе, но сохранится в истории ремонтов.
