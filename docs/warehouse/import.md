# Импорт товаров

Функция импорта позволяет массово загрузить товары в каталог из файлов CSV или Excel. Это удобно при первоначальном наполнении базы, переходе с другой системы учёта или обновлении цен.

## Когда используется

Импорт полезен, когда нужно добавить или обновить большое количество товарных позиций. Вместо ручного создания каждой карточки вы загружаете файл с данными, и система создаёт товары автоматически.

## Поддерживаемые форматы

Система поддерживает загрузку файлов в следующих форматах.

- **CSV** — текстовый формат с разделителем (точка с запятой или запятая).
- **XLSX** — формат Microsoft Excel.
- **XLS** — устаревший формат Microsoft Excel.

{% note info %}

Рекомендуется использовать формат XLSX для сохранения корректной кодировки кириллических символов. При использовании CSV убедитесь, что файл сохранён в кодировке UTF-8.

{% endnote %}

## Подготовка файла

Перед импортом подготовьте файл с данными о товарах. Каждая строка файла соответствует одному товару, каждый столбец — одному полю карточки товара.

- Первая строка файла должна содержать заголовки столбцов.
- Обязательные поля: наименование товара.
- Дополнительные поля: артикул, штрихкод, единица измерения, описание, категория, гарантия (дни), минимальный остаток, закупочная цена.

## Процесс импорта

Импорт выполняется в несколько шагов — загрузка файла, маппинг полей, предварительный просмотр и подтверждение.

### Шаг 1. Загрузка файла

Перейдите в раздел **Склад** → **Импорт** и загрузите подготовленный файл. Система автоматически определит формат и прочитает содержимое.

<!-- screenshot: форма загрузки файла для импорта -->

### Шаг 2. Маппинг полей

На этом шаге необходимо сопоставить столбцы вашего файла с полями карточки товара в системе. Система пытается сопоставить поля автоматически по заголовкам, но вы можете скорректировать маппинг вручную.

- Автоматическое сопоставление по заголовкам столбцов.
- Ручная корректировка маппинга.
- Пропуск ненужных столбцов.

<!-- screenshot: экран маппинга полей при импорте -->

{% note warning %}

Обязательно проверьте маппинг перед переходом к следующему шагу. Неправильное сопоставление полей приведёт к некорректным данным в карточках товаров.

{% endnote %}

### Шаг 3. Предварительный просмотр

Перед выполнением импорта система показывает предварительный результат — какие товары будут созданы, какие обновлены, и есть ли ошибки в данных.

- Список товаров для создания.
- Список товаров для обновления (при совпадении артикула).
- Список строк с ошибками.

### Шаг 4. Подтверждение и выполнение

После проверки предварительных результатов подтвердите импорт. Система создаст новые товары и обновит существующие.

## Обработка дубликатов

При импорте система проверяет наличие дубликатов по артикулу (SKU). Если товар с таким артикулом уже существует в каталоге, поведение определяется опцией «Обновлять существующие товары».

- **Опция включена** — данные существующего товара обновляются из файла (по совпадению артикула).
- **Опция выключена** — существующие товары пропускаются, создаются только новые.

{% note info %}

Рекомендуется включать обновление для поддержания актуальности данных и предотвращения дублирования позиций в каталоге.

{% endnote %}

## Импорт цен

При импорте можно загрузить не только основные данные товаров, но и закупочную цену. Для этого в файле должен быть столбец с заголовком «Закупочная цена», «Цена закупки» или «Цена».

- Загрузка закупочных цен вместе с товарами.
- Обновление цен у существующих товаров (при включённой опции обновления).

## Журнал импорта

После завершения импорта формируется журнал с результатами операции. В журнале отображаются созданные, обновлённые и пропущенные позиции, а также ошибки.

- Количество созданных товаров.
- Количество обновлённых товаров.
- Список ошибок с указанием номера строки.
