# Публичная проверка статуса

SanCRM предоставляет публичную страницу, на которой клиенты могут самостоятельно проверить текущий статус своего ремонта. Страница не требует авторизации и доступна по прямой ссылке или через QR-код на квитанции.

## Как это работает

Клиент переходит по ссылке или сканирует QR-код, вводит номер заказа или квитанции и получает актуальную информацию о ходе ремонта. Данные обновляются в реальном времени на основании статусов в SanCRM.

## Включение и настройка

Публичная страница проверки статуса включается в разделе настроек интеграций. После активации система генерирует уникальный URL, который можно использовать на сайте, в социальных сетях или на печатных материалах.

{% note info %}

По умолчанию публичная страница отключена. После включения она становится доступна немедленно по сгенерированной ссылке. Отключение страницы мгновенно прекращает доступ для клиентов.

{% endnote %}

<!-- screenshot: настройки публичной страницы — переключатель активации и сгенерированный URL -->

## Клиентский URL

Каждый аккаунт SanCRM получает уникальный постоянный URL для публичной страницы. Ссылку можно разместить на сайте сервисного центра, в подписи электронной почты или в мессенджерах. Адрес не меняется при обновлении настроек.

<!-- screenshot: пример ссылки на публичную страницу и кнопка копирования -->

## QR-код на квитанциях

Система автоматически генерирует QR-код, ведущий на страницу проверки статуса конкретного ремонта. QR-код можно включить в шаблон печатной квитанции, чтобы клиент мог отсканировать его камерой телефона и сразу увидеть актуальный статус.

{% note info %}

QR-код формируется индивидуально для каждого заказа и содержит прямую ссылку на страницу с предзаполненным номером. Клиенту не нужно вводить номер вручную.

{% endnote %}

<!-- screenshot: печатная квитанция с QR-кодом для проверки статуса -->

## Отображаемая информация

На публичной странице клиенту показывается ограниченный набор данных: текущий статус ремонта, тип устройства, ориентировочная дата готовности и стоимость, если она уже согласована. Внутренние комментарии и технические заметки сотрудников не отображаются.

<!-- screenshot: результат проверки статуса — карточка с информацией о ремонте на публичной странице -->

## Настройка отображаемых полей

В настройках можно выбрать, какие именно поля будут видны клиенту на публичной странице. Например, можно скрыть стоимость до момента согласования или не показывать ориентировочную дату, если она ещё не определена.

{% note info %}

Изменения в настройках отображаемых полей применяются немедленно и влияют на все публичные страницы, включая уже выданные QR-коды.

{% endnote %}

<!-- screenshot: настройки отображаемых полей публичной страницы — список полей с переключателями -->

## Брендирование страницы

Публичную страницу можно оформить в фирменном стиле сервисного центра. Доступна загрузка логотипа, выбор цветовой схемы и указание контактной информации, которая будет отображаться внизу страницы.
