# Квитанции и документы

Модуль документов обеспечивает формирование и печать всех необходимых бумаг — от приёмной квитанции до гарантийного талона. Документы генерируются на основе настраиваемых шаблонов.

<!-- screenshot: список доступных документов в карточке ремонта -->

## Типы документов

Система поддерживает несколько типов документов, необходимых в работе сервисного центра.

### Приёмная квитанция

Документ, подтверждающий приём устройства в ремонт. Выдаётся клиенту при оформлении заказа.

- Содержит информацию о клиенте, устройстве, описании неисправности и предварительной стоимости.
- Включает условия приёма и хранения устройства.
- Предусматривает место для подписи клиента.

### Акт выполненных работ

Документ, описывающий все выполненные работы и использованные запчасти. Формируется при завершении ремонта.

- Перечень выполненных услуг с ценами.
- Список использованных запчастей.
- Итоговая стоимость ремонта.

### Гарантийный талон

Документ, фиксирующий гарантийные обязательства сервисного центра.

- Гарантийные сроки на выполненные работы и установленные запчасти.
- Условия гарантийного обслуживания.
- Серийные номера установленных запчастей (при наличии).

### Акт приёма-выдачи

Документ, подтверждающий выдачу устройства клиенту после ремонта.

- Состояние устройства при выдаче.
- Подтверждение получения клиентом.
- Информация об оплате.

## Печать документов

Печать доступна из карточки ремонта. Система поддерживает несколько форматов и способов вывода.

- **Печать на принтер** — прямая печать на подключённый принтер.
- **Предварительный просмотр** — просмотр документа перед печатью.
- **Сохранение в PDF** — экспорт документа в формат PDF.
- **Печать на термопринтер** — поддержка чековых принтеров для компактных квитанций.

{% note info %}

Для печати на термопринтер необходимо настроить шаблон соответствующего формата. Подробнее в разделе [Настройки](../settings/).

{% endnote %}

## Шаблоны документов

Каждый тип документа формируется по настраиваемому шаблону. Администратор может менять содержание и оформление документов.

- **Редактор шаблонов** — визуальный редактор для создания и изменения шаблонов.
- **Переменные** — в шаблонах используются переменные, которые заменяются данными из заказа (номер, клиент, стоимость и т.д.).
- **Логотип и реквизиты** — фирменный стиль, логотип и реквизиты организации.
- **Несколько шаблонов** — для одного типа документа можно создать несколько вариантов шаблонов.

## Автоматическое формирование

Документы могут формироваться автоматически при смене статуса через [Действия Workflow](workflow-actions.md).

- Приёмная квитанция — при создании ремонта.
- Акт выполненных работ — при переходе в статус «Готов».
- Гарантийный талон — при выдаче устройства клиенту.

{% note warning %}

Убедитесь, что шаблоны документов корректно настроены и содержат все необходимые реквизиты перед началом работы. Документы с некорректными данными могут вызвать вопросы у клиентов и контролирующих органов.

{% endnote %}

## Архив документов

Все сформированные документы сохраняются в системе и доступны для повторной печати.

- История всех напечатанных документов по ремонту.
- Возможность повторной печати в любой момент.
- Документы сохраняют данные на момент формирования, даже если карточка ремонта была изменена позднее.
