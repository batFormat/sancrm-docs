# Действия Workflow

Действия Workflow — автоматические операции, которые выполняются при смене статуса ремонта. Они позволяют автоматизировать рутинные задачи и исключить человеческий фактор.

<!-- screenshot: список настроенных workflow-действий в настройках -->

## Принцип работы

Workflow-действия привязываются к переходам между статусами. Когда ремонт переходит из одного статуса в другой, система автоматически выполняет все связанные с этим переходом действия.

- Действие срабатывает после успешной смены статуса.
- Одному переходу можно назначить несколько действий.
- Действия выполняются в заданном порядке.

## Типы действий

Система поддерживает несколько типов автоматических действий, покрывающих основные потребности сервисного центра.

### Отправка уведомления

Автоматическая отправка SMS, email или Telegram-сообщения клиенту при смене статуса.

- Выбор канала уведомления (SMS, email, Telegram).
- Использование шаблонов с переменными (имя клиента, номер заказа, статус).
- Настройка задержки перед отправкой.

### Назначение исполнителя

Автоматическое назначение мастера на ремонт по заданным правилам.

- Назначение конкретного сотрудника.
- Назначение по принципу наименьшей загрузки.
- Назначение на основе типа устройства или неисправности.

### Списание запчастей

Автоматическое списание добавленных запчастей со склада при переходе ремонта в определённый статус.

- Списание происходит при переходе в статус «Готов» или другой настроенный статус.
- Проверка наличия запчастей на складе перед списанием.
- Создание складской операции с привязкой к заказу.

### Формирование документа

Автоматическая генерация документа при смене статуса.

- Создание акта выполненных работ при переходе в «Готов».
- Формирование гарантийного талона при выдаче.
- Документ доступен для печати из карточки ремонта.

{% note info %}

Шаблоны документов настраиваются в разделе [Настройки](../settings/). Вы можете создать собственные шаблоны с фирменным стилем вашего сервисного центра.

{% endnote %}

### Создание задачи

Автоматическое создание задачи для сотрудника при смене статуса.

- Задача назначается на указанного сотрудника или роль.
- Настраивается текст задачи и срок выполнения.
- Задача отображается в личном списке дел сотрудника.

### Изменение поля

Автоматическое изменение значения поля в карточке ремонта.

- Установка даты готовности при переходе в «В работе».
- Фиксация даты выдачи при переходе в «Выдан».
- Изменение любого пользовательского поля.

## Настройка действий

Настройка workflow-действий выполняется администратором в разделе настроек.

<!-- screenshot: форма настройки нового workflow-действия -->

- **Выбор перехода** — указание пары статусов «из — в», при которой срабатывает действие.
- **Тип действия** — выбор одного из поддерживаемых типов.
- **Параметры** — настройка конкретных параметров действия (шаблон, получатель, поле).
- **Условия** — дополнительные условия срабатывания (например, только для срочных ремонтов).

{% note warning %}

Тестируйте workflow-действия на тестовых ремонтах перед применением к реальным заказам. Некорректно настроенное действие может привести к ошибочным уведомлениям клиентов или списанию запчастей.

{% endnote %}

## Журнал выполнения

Все выполненные workflow-действия записываются в журнал. Это помогает отслеживать автоматические операции и диагностировать проблемы.

- Дата и время выполнения.
- Тип и параметры действия.
- Результат — успех или ошибка с описанием причины.
- Связанный ремонт и переход.
