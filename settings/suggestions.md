# Справочники подсказок

Справочники подсказок содержат предзаполненные варианты для полей карточки ремонта — типы дефектов, описания внешнего вида, комплектность устройства и другие параметры. Подсказки ускоряют оформление заказов и стандартизируют заполнение данных.

<!-- screenshot: страница управления справочниками подсказок -->

## Типы дефектов

Справочник типов дефектов содержит стандартные формулировки неисправностей, которые предлагаются сотруднику при заполнении поля «Заявленная неисправность» в карточке ремонта.

- Примеры: «Не включается», «Разбит экран», «Не заряжается», «Залит жидкостью», «Не работает камера».
- При вводе текста система предлагает подходящие варианты из справочника.

<!-- screenshot: справочник типов дефектов — список записей -->

### Добавление типа дефекта

Для добавления новой записи введите текст дефекта и, при необходимости, привяжите его к категории устройств. Привязка к категории позволяет показывать только релевантные подсказки — например, дефект «Не работает тачпад» будет предлагаться только для ноутбуков.

<!-- screenshot: форма добавления типа дефекта -->

## Описание внешнего вида

Справочник описаний внешнего вида помогает стандартизировать фиксацию состояния устройства при приёме. Подсказки описывают типичные дефекты корпуса и экрана.

- Примеры: «Без видимых повреждений», «Царапины на корпусе», «Скол на углу», «Трещина на экране», «Потёртости на задней крышке».
- Выбранные описания автоматически подставляются в поле внешнего вида карточки ремонта.

{% note info %}

Точное описание внешнего вида при приёме защищает сервисный центр от претензий клиента о повреждениях, якобы полученных во время ремонта.

{% endnote %}

<!-- screenshot: справочник описаний внешнего вида -->

## Чек-листы комплектности

Чек-лист комплектности определяет, что принимается вместе с устройством: зарядное устройство, кабель, чехол, коробка и другие аксессуары. При оформлении ремонта приёмщик отмечает наличие каждого пункта.

- Примеры: «Зарядное устройство», «USB-кабель», «Чехол», «Защитное стекло», «Коробка», «Документы».
- Чек-лист можно привязать к категории устройства — для смартфонов и ноутбуков набор аксессуаров будет разным.

<!-- screenshot: чек-лист комплектности с отмеченными пунктами -->

{% note warning %}

Тщательная проверка комплектности при приёме предотвращает споры при выдаче устройства. Зафиксированный чек-лист фиксируется в карточке ремонта и распечатывается в квитанции.

{% endnote %}

## Пользовательские справочники

Помимо встроенных справочников, вы можете создать собственные словари для произвольных полей. Пользовательский справочник привязывается к конкретному полю и предлагает варианты при его заполнении.

### Создание справочника

Укажите название справочника и поле, к которому он привязан. Затем добавьте варианты значений — каждый на отдельной строке.

<!-- screenshot: форма создания пользовательского справочника -->

### Примеры использования

- **Способ обращения** — «Рекомендация», «Реклама в интернете», «Проходил мимо», «Постоянный клиент».
- **Срочность** — «Обычная», «Срочная», «Экспресс».
- **Тип оплаты** — «Наличные», «Карта», «Перевод», «Рассрочка».

<!-- screenshot: пример пользовательского справочника «Способ обращения» -->

## Импорт и экспорт

Записи справочника можно импортировать из текстового файла (каждая строка — отдельное значение) или экспортировать для резервного копирования. Это полезно при переносе данных из другой системы.
