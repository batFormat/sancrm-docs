# Пользователи и роли

Управление пользователями и ролями позволяет контролировать, кто имеет доступ к системе и какие действия может выполнять каждый сотрудник.

<!-- screenshot: страница управления пользователями — список сотрудников -->

## Приглашение пользователей

Для добавления нового сотрудника в систему отправьте ему приглашение по email. Пользователь получит ссылку для регистрации и автоматически попадёт в вашу организацию.

- Укажите email-адрес сотрудника и выберите роль, которая будет назначена после регистрации.
- Приглашение действительно ограниченное время — если сотрудник не зарегистрировался, приглашение можно отправить повторно.

<!-- screenshot: форма приглашения нового пользователя -->

## Система ролей

SanCRM использует ролевую модель для разграничения доступа. Каждому пользователю назначается одна или несколько ролей, определяющих набор доступных функций.

### Встроенные роли

Система предоставляет набор предустановленных ролей, покрывающих типичные сценарии работы сервисного центра.

- **Администратор** — полный доступ ко всем разделам, включая настройки системы и управление пользователями.
- **Мастер** — работа с ремонтами, смена статусов, добавление запчастей и услуг.
- **Приёмщик** — создание ремонтов, приём и выдача устройств, работа с клиентской базой.
- **Кассир** — проведение оплат, работа с кассовыми операциями.

<!-- screenshot: список встроенных ролей с описанием -->

### Пользовательские роли

При необходимости можно создать собственные роли с индивидуальным набором прав. Это полезно, когда стандартные роли не соответствуют структуре вашей организации.

<!-- screenshot: форма создания пользовательской роли -->

## Настройка прав доступа

Каждая роль содержит набор разрешений, сгруппированных по модулям системы. Права определяют доступ к просмотру, созданию, редактированию и удалению данных.

- **Ремонты** — создание, редактирование, удаление, смена статуса, назначение исполнителя.
- **Склад** — просмотр остатков, создание документов, проведение инвентаризации.
- **Клиенты** — просмотр и редактирование клиентской базы.
- **Финансы** — доступ к кассовым операциям и финансовым отчётам.
- **Настройки** — доступ к разделу настроек системы.

{% note info %}

Изменение прав роли применяется ко всем пользователям с этой ролью немедленно. Перезагрузка страницы не требуется.

{% endnote %}

## Деактивация пользователей

Если сотрудник больше не работает в вашей организации, его учётную запись можно деактивировать. Деактивированный пользователь не может войти в систему, но его история действий сохраняется.

{% note warning %}

Деактивация не удаляет пользователя из системы. Все ремонты и действия, привязанные к этому сотруднику, остаются в истории. При необходимости учётную запись можно активировать повторно.

{% endnote %}

<!-- screenshot: кнопка деактивации в карточке пользователя -->

## Журнал действий пользователей

Система ведёт журнал действий всех пользователей: входы в систему, создание и редактирование записей, изменение статусов. Журнал доступен администраторам и помогает отслеживать работу сотрудников.
