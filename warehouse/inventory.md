# Инвентаризация

Документ инвентаризации позволяет сверить фактические остатки товаров на складе с учётными данными в системе. По результатам сверки система автоматически корректирует остатки — оприходует излишки и списывает недостачу.

## Когда используется

Инвентаризация проводится для контроля достоверности складского учёта. Рекомендуется выполнять её регулярно — ежемесячно или ежеквартально, а также при смене материально ответственного лица.

## Создание документа инвентаризации

Для создания инвентаризации перейдите в раздел складских документов и выберите тип «Инвентаризация». Укажите склад и начните заполнение фактических остатков.

<!-- screenshot: форма создания документа инвентаризации -->

### Основные поля

- **Склад** — склад, на котором проводится инвентаризация.
- **Дата** — дата проведения.
- **Ответственный** — сотрудник, проводящий инвентаризацию.
- **Комментарий** — дополнительная информация.

{% note warning %}

Во время проведения инвентаризации рекомендуется приостановить другие складские операции на данном складе, чтобы избежать расхождений.

{% endnote %}

## Заполнение фактических остатков

В документ инвентаризации загружаются все товары, числящиеся на выбранном складе. Для каждой позиции необходимо указать фактическое количество.

- Автоматическая загрузка позиций с текущими учётными остатками.
- Ввод фактического количества вручную.
- Сканирование штрихкодов для подсчёта.

<!-- screenshot: таблица инвентаризации с колонками «Учёт» и «Факт» -->

## Сканирование штрихкодов

Для ускорения подсчёта можно использовать сканер штрихкодов. При каждом сканировании система увеличивает фактическое количество соответствующего товара на единицу.

- Подключение сканера штрихкодов.
- Автоматическое увеличение счётчика при сканировании.
- Звуковое подтверждение успешного сканирования.

{% note info %}

Сканер штрихкодов подключается как обычная клавиатура (USB HID). Специальная настройка драйверов не требуется.

{% endnote %}

## Расхождения

После заполнения фактических остатков система автоматически рассчитывает расхождения. Расхождения отображаются по каждой позиции и подсвечиваются цветом.

- **Излишек** — фактическое количество больше учётного. Товар будет оприходован.
- **Недостача** — фактическое количество меньше учётного. Товар будет списан.
- **Совпадение** — фактическое и учётное количество совпадают.

<!-- screenshot: расхождения в документе инвентаризации с цветовой индикацией -->

## Проведение документа

При проведении документа инвентаризации система автоматически корректирует остатки на складе. Излишки приходуются, недостачи списываются.

- Автоматическая коррекция остатков.
- Формирование итогового отчёта по инвентаризации.
- Фиксация всех расхождений в истории.

{% note warning %}

Проведение инвентаризации необратимо корректирует остатки. Внимательно проверьте все расхождения перед проведением документа.

{% endnote %}
