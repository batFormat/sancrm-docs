# Типы цен

Система поддерживает несколько типов цен для каждого товара. Это позволяет гибко управлять ценообразованием — задавать разные цены для розничных продаж, оптовых клиентов и ремонтных услуг.

## Назначение

Типы цен позволяют вести многоуровневое ценообразование. Каждому товару можно присвоить несколько цен, которые используются в разных сценариях — при продаже в розницу, отпуске оптовому клиенту или включении запчасти в стоимость ремонта.

## Предустановленные типы цен

Система содержит базовые типы цен, которые доступны сразу после настройки.

- **Закупочная цена** — цена, по которой товар был приобретён у поставщика (себестоимость).
- **Розничная цена** — цена для конечного покупателя при продаже через кассу или витрину.
- **Оптовая цена** — цена для оптовых клиентов, как правило, ниже розничной.
- **Ремонтная цена** — цена, по которой запчасть включается в стоимость ремонта.

## Создание типа цен

Для создания нового типа цен перейдите в настройки ценообразования. Укажите название и способ расчёта.

- **Название** — отображаемое имя типа цен.
- **Способ расчёта** — фиксированная цена или наценка от базовой.

<!-- screenshot: форма создания нового типа цен -->

## Правила наценки

Цена может рассчитываться автоматически на основе наценки от закупочной цены или другого типа цен. Это избавляет от необходимости вручную пересчитывать цены при изменении себестоимости.

- Наценка в процентах от закупочной цены.
- Наценка в процентах от другого типа цен.
- Фиксированная надбавка в денежном выражении.
- Округление до целых значений.

### Пример расчёта

| Закупочная цена | Наценка | Розничная цена |
|---|---|---|
| 1 000 руб. | 50% | 1 500 руб. |
| 500 руб. | 50% | 750 руб. |
| 2 300 руб. | 30% | 2 990 руб. (с округлением) |

{% note warning %}

При изменении закупочной цены (например, при новом приходе) автоматически пересчитанные цены обновятся только после проведения документа прихода. Фиксированные цены не пересчитываются автоматически.

{% endnote %}

## Назначение цен товарам

Цены устанавливаются в карточке товара или массово через импорт. Для каждого товара можно задать значения по каждому типу цен.

- Ручная установка цены в карточке товара.
- Массовое обновление цен через импорт.
- Автоматический расчёт по правилам наценки.

<!-- screenshot: вкладка «Цены» в карточке товара -->

## Использование типов цен

Типы цен применяются в различных сценариях работы системы.

- **Продажа через кассу** — используется розничная цена.
- **Оформление ремонта** — используется ремонтная цена для запчастей.
- **Отчёты** — анализ маржинальности на основе закупочной и продажной цен.
