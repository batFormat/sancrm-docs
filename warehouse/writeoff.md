# Списание

Документ списания фиксирует удаление товаров со склада. При проведении списания остатки на выбранном складе уменьшаются. Списание используется, когда товар повреждён, утерян или непригоден к продаже.

## Когда используется

Списание применяется в ситуациях, когда товар необходимо удалить с учёта без продажи или передачи. Типичные причины — брак, истечение срока годности, порча при хранении или утеря.

## Создание документа списания

Для создания списания перейдите в раздел складских документов и выберите тип «Списание». Укажите склад, причину и добавьте позиции.

<!-- screenshot: форма создания документа списания -->

### Основные поля

- **Склад** — с какого склада списываются товары.
- **Причина списания** — основание для удаления товаров с учёта.
- **Дата** — дата проведения списания.
- **Комментарий** — дополнительная информация.

## Причины списания

При создании документа необходимо указать причину списания. Это помогает анализировать потери и выявлять системные проблемы.

- Брак / повреждение.
- Истечение срока годности.
- Утеря.
- Использование для нужд компании.
- Прочее.

{% note info %}

Список причин списания можно расширить в настройках системы, добавив собственные варианты.

{% endnote %}

## Добавление товарных позиций

В документ списания добавляются товары с указанием количества, подлежащего списанию. Количество не может превышать текущий остаток на выбранном складе.

- Поиск товара по наименованию, артикулу или штрихкоду.
- Указание количества для списания.
- Контроль доступного остатка.

<!-- screenshot: таблица позиций в документе списания -->

{% note warning %}

Нельзя списать больше товаров, чем имеется на складе. Если остатки недостаточны, проведение документа будет заблокировано.

{% endnote %}

## Выбор склада

Товары списываются с конкретного склада. Убедитесь, что выбран правильный склад, так как остатки будут уменьшены именно на нём.

- Выбор склада из списка доступных.
- Отображение текущих остатков выбранного склада.

## Согласование

В зависимости от настроек системы списание может требовать согласования руководителем. В таком случае документ остаётся в статусе «На согласовании» до подтверждения.

- Автоматическая отправка на согласование.
- Уведомление ответственного лица.
- Подтверждение или отклонение списания.

{% note info %}

Настройка обязательного согласования списания выполняется в разделе администрирования.

{% endnote %}

## Проведение документа

После заполнения и (при необходимости) согласования документ проводится. Остатки уменьшаются на указанное количество.

- Автоматическое обновление остатков.
- Фиксация даты и автора списания.
